01 · Checkbox / cyber-tick — the flagship form control

unchecked hover focus checked indeterminate
Cyber-tick. The marker is a chamfered slab — corner cut on the bottom-left and top-right reads as forward motion. The check itself is an angular polygon, not a stroked SVG path; on :checked the box fills with the active accent and the tick is painted in the surface colour so it punches through the fill. Three intent tints — cyan (default), lime (always-on / passive), warm (info / locked) — keep semantic meaning legible.
DEFAULT — cyan
LIME — always-on / passive bonus
WARM — info / linked / installed
WARN / ERROR — destructive intent
SIZE — sm / default / lg

The chamfer scale is bound to the box dimension — --check-cut grows with --check-size so the silhouette stays in proportion across 16 → 26px. Indeterminate state paints a horizontal bar (same painting strategy as the tick) so screens of partial-selection trees still read as a coherent shape language.

02 · Anatomy — what the chamfer is doing

OUTER — chamfered slab. clip-path polygon with TR + BL corners cut.
1px FAKE BORDER — outer painted in the accent colour; ::before inset by 1px paints the inner fill. Real border gets clipped away by clip-path, so the gap reads as the stroke.
FILL — on :checked, the inner ::before swaps to the accent colour. No animation.
TICK — polygon, not SVG. The corners of the check are sharp angles, matching the shape vocabulary. Painted in --cpb-bg-solid-top so it reads against the bright fill.
FOCUS — frame swaps to accent. No outline ring (rings get clipped).
